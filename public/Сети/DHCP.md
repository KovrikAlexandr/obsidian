Dynamic Host Configuration Protocol – протокол динамической выдачи узлу его сетевых настроек.

Сетевые настройки:
- IP адрес в сети
- Маска подсети (Subnet Mask)
- Адрес DNS
- Default gateway

Есть клиент – узел, который хочет получить сетевые настройки
Есть DHCP-сервер – единый узел, который выдаёт настройки

### Процесс выдачи
Выдача сетевой конфигурации происходит в 4 этапа:
#### 1. DHCP Discover
У клиента ещё нет адреса.
Клиент отправляет широковещательный запрос, чтобы понять, кто может выдать ему конфигурацию сети.

#### 2. DHCP Offer
DHCP принял discover и отправляет широковещательный (у клиента ещё нет адреса) запрос, предлагая конфигурацию

Если в сети несколько DHCP серверов, то каждый получивший рассылку от клиента отправит offer.

#### 3. DHCP Request
Клиент соглашается с конфигурацией одного из серверов и отправляет широковещательный запрос.

Таким образом сервер, чью конфигурацию приняли фиксирует этот адрес. Остальные сервера принимают отказ.

#### 4. DHCP Ack
Сервер подтверждает выделение адреса и отправляет окончательный набор параметров.  
После этого клиент может использовать свой IP.


### Важные детали

#### Lease
Адрес выдаётся не навсегда. Адрес выдаётся DHCP сервером на определённый срок – lease. По истечении половины срока клиент запрашивает продление срока аренды

#### Relay
Маршрутизация объединяет несколько локальных сетей, но широковещательные запросы отправляются только в рамках одной локальной сети и выйти из неё не могут.

Это создаёт проблему, если у нас есть один DHCP сервер на несколько локальных сетей
